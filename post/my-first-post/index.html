How to Easily Build a Blog Using Hugo and Github Pages

This is my first blog trip.

In this post, I’d like to share the specific steps that I performed to build my personal blog using Hugo and Github Pages.

And the article refers to many previous tutorials on the Internet. Let’s start!

​

Step 1: Install Hugo

  1. Refer to the official Hugo installation guide and get prepared.

    Get a basic knowledge and install the necessary tool (such as git-scm.com/ 、golang.org/dl/) for your development environment if you lack.

  2. Download the appropriate version for your platform from Hugo Releases.

  • If you are on macOS and using Homebrew, you can install Hugo with the following one-liner:

    brew install hugo

  • If you are on a Windows machine,you can download the latest zipped Hugo executable from Hugo Releases.

 a. Move the ZIP file into your `D:\Hugo\bin` folder.

 b. Double-click on the ZIP file and extract its contents. Be sure to extract the contents into the same ` D:\Hugo\bin` folder.

 c. You should now have three new files: The hugo executable (`hugo.exe`), `LICENSE`, and `README.md`. And use the command `set PATH=%PATH%;C:\Hugo\bin`.
  1. After installation, run the following command to verify that hugo runs correctly.
hugo version

Step 2: Create a new Hugo site

  1. Go to the directory where you want to place the Hugo site folder.

  2. Run the following command to create a new Hugo site.

hugo new site myblog # "myblog" is my site folder name.

Running the hugo new site generator from the command line will create a directory structure with the following elements:

.
├── archetypes
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

Step 3: Clone a theme

  1. Go to the Hugo Themes page and choose one theme for your site.Take the Ananke theme as an example in the following text.

  2. Clone it to a local directory by running the following command.

cd myblog
git clone https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

Step 4: Create a new post

  1. Go to the root directory of your site folder.

    cd myblog  # "myblog" is the site folder name.

  2. Create a new post using the following command.

    hugo new post/my-first-post.md  # "my-first-post.md" is the file name of the new post. I also created such a file for testing.

  3. Edit the newly created post and add some content.

    Step 5: Edit the configuration file

  4. In the root directory of your site folder, refer to the configuration explanation of your theme and edit the config.toml file according to your need.

hugo server --theme=ananke --buildDrafts

The theme configuration item is the name of your site theme, which must be the same as the name of the directory that your theme is cloned to. In this example, it is theme = "ananke".

  1. Preview your new site at http://localhost:1313/ using a browser.

If you’re satisfied with the preview, move to the next step. If not, edit your post or the config.toml file and preview again.

Step 6: Create a Github repository

  1. Create a Github account.
    • If you already have one, sign in and skip to the next step.
    • If you don’t have one, go to GitHub, sign up an account and sign in.
  2. Go to GitHub Pages, refer to the easy-to-use guide for User or organization site.
  3. Create a new repository named username.github.io, where username is your username on GitHub.

Step 7: Build your Hugo site

In the root directory of your site folder, run the Hugo command to build your site:

hugo -t ananke
hugo --theme=ananke --baseUrl="https://github.com/Hyacinthyue/Hyacinthyue.github.io.git/" --buildDrafts
# Builds your Hugo site and writes your static site to the "public" directory by default.

The default Hugo target directory for your built website is public/. However, you can change this value by specifying a different publishDir in your site configuration.

Step 8: Convert your site folder to a Git repository

  1. Go to the myblog/public directory and initialize the Git repository.
cd myblog/public
git init  # Initializes the Git repository.
git add . # Adds all modified or newly created files. You can also just add a certain file.
git commit -m "Add a new post"  # "Add a new post" is the commit message.
  1. Check the files in the public directory using the following command. You’ll find the .git file is added.

Step 9: Link the local repository to the remote origin

  1. In the myblog/public directory, add the remote origin for your local Git repository using the following command.

    git remote add origin https://github.com/Hyacinthyue/Hyacinthyue.github.io.git/
# "https://github.com/Hyacinthyue/Hyacinthyue.github.io.git" indicates "https://github.com/your-github-id/your-github-id.github.io.git".

  1. Push your changes to the remote origin.

    git push -u origin master

    Now you’ve got your customized personal blog! For me, it is at https://lilin90.github.io.
